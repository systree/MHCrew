================================================================
MOVER HERO — WEBHOOK TEST PAYLOADS
================================================================
Endpoint (POST all to this URL):
  http://localhost:3001/api/webhooks/ghl

Headers (all requests):
  Content-Type: application/json

Fire in this exact order:
  1. INSTALL
  2. UserCreate
  3. OpportunityCreate   ← needs a real GHL opportunity ID (see note)
  4. OpportunityAssignedToUpdate

================================================================
IMPORTANT NOTES
================================================================
- Keep locationId IDENTICAL across all 4 payloads.
- The "id" in UserCreate must match "assignedTo" in
  OpportunityAssignedToUpdate.
- The "id" in OpportunityCreate must match "id" in
  OpportunityAssignedToUpdate.
- OpportunityCreate triggers a live GHL API call using your
  n8n token. Use a REAL opportunity ID from your GHL account.
  If the ID is fake the job upsert will fail (fetchFullOpportunity
  returns null) — all other events are safe to test with dummy IDs.
- Use a phone number in UserCreate that is in OTP_BYPASS_PHONES
  so you can log in to the app afterwards to see the job.

CHECKING RESULTS IN SUPABASE

FROM mh_pwa_ghl_sync_log
ORDER BY created_at DESC;
